# Changing Style via CSS


Updating the CSS allows you to change the UI of our components to meet your needs.

This section will guide you through an example of updating the look of buttons. Note that the process can be applied to update any HTML tag from within SuperTokens components.

## Global style changes

### How to make changes

First, let's open our website at `/auth`. The Sign-in widget should show up. Let's use the browser console to find out the class name that we'd like to overwrite.


<img alt="Inspecting submit button in prebuilt form" width="500px" src="/img/emailpassword/inspect-button.png" />

<img alt="Highlighting attribute to be used for customization" width="500px" src="/img/emailpassword/inspect-button-2.png" />

Each stylable components contains `data-supertokens` attributes (in our example `data-supertokens="button"`). 

Let's add a `button` attribute to our configuration file. The syntax for styling is the same as [React inline styling](https://www.w3schools.com/react/react_css.asp).

<FrontendSDKTabs>
<TabItem value="reactjs">

```tsx
import SuperTokens from "supertokens-auth-react";
import EmailPassword from "supertokens-auth-react/recipe/emailpassword";
import Session from "supertokens-auth-react/recipe/session";

SuperTokens.init({
    appInfo: {
        apiDomain: "...",
        appName: "...",
        websiteDomain: "..."
    },
    recipeList: [
        EmailPassword.init({
            palette: { /* ... */ },
            // highlight-start
            style: {
                button: {
                    backgroundColor: '#252571',
                    border: '0px',
                    width: '30%',
                    margin: '0 auto'
                }
            }
            // highlight-end
        }),
        Session.init()
    ]
});
```
</TabItem>
</FrontendSDKTabs>

### Changing fonts

By default, SuperTokens loads and uses the ['Rubik'](https://fonts.google.com/specimen/Rubik) font. The best way to override this is to add styling to the `container` component in the recipe configuration. Doing so will prevent the SDK from loading the default font.

<FrontendSDKTabs>
<TabItem value="reactjs">

```tsx
import SuperTokens from "supertokens-auth-react";
import EmailPassword from "supertokens-auth-react/recipe/emailpassword";
import Session from "supertokens-auth-react/recipe/session";

SuperTokens.init({
    appInfo: {
        apiDomain: "...",
        appName: "...",
        websiteDomain: "..."
    },
    recipeList: [
        EmailPassword.init({
            palette: { /* ... */ },
            // highlight-start
            style: {
                container: {
                    fontFamily: "cursive"
                }
            }
            // highlight-end
        }),
        Session.init()
    ]
});
```
</TabItem>
</FrontendSDKTabs>

### Using media queries

You may want to have different CSS for different viewports. This can be achieved via media queries like this:

<FrontendSDKTabs>
<TabItem value="reactjs">

```tsx
import SuperTokens from "supertokens-auth-react";
import EmailPassword from "supertokens-auth-react/recipe/emailpassword";

SuperTokens.init({
    appInfo: {
        apiDomain: "...",
        appName: "...",
        websiteDomain: "...",
    },
    recipeList: [
        // ...
        EmailPassword.init({
            // ...
            style: {
                button: {
                    backgroundColor: '#252571',
                    border: '0px',
                    width: '30%',
                    margin: '0 auto',
                    // highlight-start
                    "@media (max-width: 440px)": {
                        width: "90%",
                    },
                    // highlight-end
                },
            },
        }),
    ],
});
```
</TabItem>
</FrontendSDKTabs>
